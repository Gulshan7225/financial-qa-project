# API Documentation

Base URL (local): `http://localhost:8000/api/v1`

Interactive docs are auto-generated by FastAPI at `/docs` (Swagger) and
`/redoc` once the server is running.

Auth: if `FINQA_API_KEY` is set in `.env`, every endpoint below requires
header `X-API-Key: <your key>`. If unset, no auth is required.

---

## POST /reports/upload

Upload a PDF financial report. Validates the file and runs the full
extraction pipeline (text, tables, financial facts, charts).

**Request:** `multipart/form-data`, field `file` (PDF only)

**Response `200`**
```json
{
  "report_id": "6e2f...c1a4",
  "filename": "Acme_Quarterly_Report.pdf",
  "status": "done",
  "message": "File uploaded and processed successfully."
}
```

**Errors**
- `415` — not a PDF (wrong content-type or extension)
- `400` — file content doesn't have a valid PDF signature
- `413` — file exceeds size limit

## GET /reports/{report_id}/tables

Returns every table detected in the report, with headers and rows exactly
as parsed. For a company financial report this is the income
statement/balance sheet; for a CAMS/KFintech mutual-fund Consolidated
Account Summary, this is the holdings table (Folio No, ISIN, Scheme Name,
Cost Value, Unit Balance, NAV Date, NAV, Market Value, Registrar) — the
document type is auto-detected, no separate endpoint needed.

```json
[
  {
    "table_id": "a1b2c3d4",
    "page": 2,
    "title": "Income Statement",
    "headers": ["Line Item", "Q1 FY24", "Q2 FY24", "Q3 FY24", "Q4 FY24"],
    "rows": [
      ["Total Revenue", "$42.5M", "$47.1M", "$51.8M", "$58.3M"],
      ["Net Income", "$7.8M", "$9.1M", "$10.9M", "$13.2M"]
    ]
  }
]
```

## GET /reports/{report_id}/charts

Returns detected chart images with a caption and (if an LLM key is
configured) a best-effort structured data series.

```json
[
  {
    "chart_id": "20ba4ecd",
    "page": 1,
    "description": "Bar chart showing quarterly revenue growth from $42.5M to $58.3M across FY24.",
    "extracted_series": {
      "chart_type": "line",
      "x_label": "Quarter",
      "y_label": "Revenue ($M)",
      "series": [{"label": "Revenue", "points": [{"x": "Q1 FY24", "y": 42.5}]}]
    },
    "confidence": "medium"
  }
]
```

## GET /reports/{report_id}/charts/{chart_id}/image

Returns the raw cropped chart image (`image/png`).

## GET /reports/{report_id}/summary

Computed key metrics (latest value, full history, period-over-period
growth %) derived directly from the extracted facts, plus computed gross
and net margins.

```json
{
  "report_id": "6e2f...",
  "key_metrics": {
    "revenue": {
      "latest_value": 58300000.0,
      "latest_period": "Q4 FY24",
      "unit": "$",
      "history": [{"period": "Q1 FY24", "value": 42500000.0}, ...],
      "period_over_period_growth_pct": 12.55
    },
    "net_margin_pct_computed": {"latest_value": 22.64}
  },
  "period_covered": ["FY23", "FY24", "Q1 FY24", "Q2 FY24", "Q3 FY24", "Q4 FY24"],
  "notes": ["Values are extracted directly from tables detected in the PDF.", "..."]
}
```

## POST /reports/{report_id}/query

Ask a natural-language question about the report.

**Request body**
```json
{ "question": "What was total revenue in Q4 FY24?", "top_k": 5 }
```

**Response**
```json
{
  "report_id": "6e2f...",
  "question": "What was total revenue in Q4 FY24?",
  "answer": "Revenue for Q4 FY24 was $58.30M (source: page 2, extracted table).",
  "confidence": "high",
  "matched_facts": [
    {"metric": "revenue", "period": "Q4 FY24", "value": 58300000.0, "unit": "$", "source_type": "table", "source_id": "a1b2c3d4", "page": 2}
  ],
  "sources": [
    {"source_type": "table", "page": 2, "reference": "Table on page 2", "snippet": "Total Revenue: 58300000.0"}
  ]
}
```

`confidence` is `"high"` for a direct table-lookup answer, `"medium"`/`"low"`
for retrieval-based answers depending on match quality and whether an LLM
was used to phrase the answer.

**Errors**
- `409` — report hasn't finished processing yet
- `404` — `report_id` not found

## GET /reports/{report_id}/chart-image?metric=revenue

Renders a PNG bar chart of a single metric's trend across reported
periods, generated directly from the extracted facts.

Valid `metric` values: `revenue`, `cost_of_goods_sold`, `gross_profit`,
`operating_expenses`, `operating_income`, `ebitda`, `net_income`,
`total_assets`, `total_liabilities`, `total_equity`,
`cash_and_equivalents`, `eps`, `total_debt` (whichever were actually
detected in the uploaded report — check `/summary` for what's available).

## GET /reports/{report_id}/compare-image?metrics=revenue,net_income

Renders a grouped bar chart comparing several metrics across shared
periods.

## DELETE /reports/{report_id}

Deletes the uploaded file, cropped chart images, and all extracted data
for a report.

```json
{ "report_id": "6e2f...", "deleted": true }
```
